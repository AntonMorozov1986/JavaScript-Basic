Упражнение №1. Разбор кода.


Дан код:
1 -- var a = 1, b = 1, c, d;
2 -- c = ++a; alert(c); // 2
3 -- d = b++; alert(d); // 1
4 -- c = (2+ ++a); alert(c); // 5
5 -- d = (2+ b++); alert(d); // 4
6 -- alert(a); // 3
7 -- alert(b); // 3


Разбор кода:

1 -- В первой строке мы объявляем переменные "a", "b", "c" и "d". Переменным "a" и "b" присваиваем значения 1;

2 -- Во второй строке мы выполняем инкремент в префиксной форме (возвращаем значение после увеличения переменной на 1) переменной "a" и присваиваем значение переменной "c". Таким образом выражение "++a" выполнит инкремент переменной "a" и вернет значение 2, которое мы присвоим переменной "c". И выведем значение "с", которое стало равным 2.

3 -- В третьей строке мы выполняем инкремент в постфиксной форме (возвращаем значение переменной, а после увеличиваем её значение на 1) переменной "b" и присваиваем значение переменной "d". Таким образом выражение "b++" вернет значение переменной "b" (значение равно 1) и после выполнит инкремент переменной "b", Значение, которое вернуло выражение "b++" присвоим переменной "d". И выведем значение "d", которое стало равным 1.

4 -- В четвертой строке кода мы выполняем в первую очередь инкремент в префиксной форме переменной "a", т.к. приоритет инкремента 17, а бинарного "+" 13, выражение вернет значение 3 (три), т.к. переменная "a" после второй строки стало равно 2 и мы увеличили значение еще на 1. Далее выполняем операцию сложения - (2 + 3) и получаем результат 5, который присваиваем перменной "c". Значение переменной "с" выводим на экран.

5 -- В пятой строке кода мы выполняем инкремент в постфиксной форме переменной "b" выражение вернет значение 2 (два), т.к. переменная "b" после третьей строки стало равно 2, а увеличение значение будет выполнено после возврата значения переменной "b". Таким образом выражение "b++" вернет значение 2, а после увеличит значение переменной "b" на 1, которое станет равное 3. Далее выполняем операцию сложения - (2 + 2) и получаем результат 4, который присваиваем перменной "d". Значение переменной "d" выводим на экран.

6 -- В шестой и седьмой строках мы выводим значения переменных "a" и "b". Значения переменных равно 3 (трем), т.к. мы выполнили инкремент этих переменных 2 раза.

Упражнение №7. Сравнение null и "0".


alert( null > 0 ); // (1) false
alert( null == 0 ); // (2) false
alert( null >= 0 ); // (3) true

С точки зрения математики это странно. Результат последнего сравнения говорит о том, что "null больше или равно нулю", тогда результат одного из сравнений выше должен быть true, но они оба ложны.
Причина в том, что нестрогое равенство (==) и сравнения (>, <, >=, <=) работают по-разному. Сравнения преобразуют null в число, рассматривая его как 0. Поэтому выражение (3) null >= 0 истинно, а null > 0 ложно.
С другой стороны, для нестрогого равенства == значений undefined и null действует особое правило: эти значения ни к чему не приводятся, они равны друг другу и не равны ничему другому. Поэтому (2) null == 0 ложно.
